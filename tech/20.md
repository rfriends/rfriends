### 20.1 遅延パラメータの調整  
  
> [!CAUTION]
> ただし、ほかの録音がない時間帯に行ってください。
  
リアルタイム録音で開始が遅い、最後が切れている場合、  
以下の調整を行ってみてください。  
  
1) delay  
実際の放送から何秒遅れて配信されているかを設定  
 
3) pre_margin  
配信の前に何秒余裕を持たせて録音するかの設定
  
5) post_margin  
配信の後に何秒余裕を持たせて録音するかの設定

以下の場所に設定パラメータがあります。  
  
```  
メニュー - 設定 - パラメータ - 編集  
  
-------------------------------------------------  
radiko premium (338行付近)  
  
; 録音開始 : 番組開始時間 + deley - pre_mergin  
; 録音終了 : 番組終了時間 + deley + post_mergin  
;  
;　配信遅れ(0 - 120 sec)  
premium_delay = 40  
;　前の余裕(0 - 120 sec)  
premium_pre_margin = 25  
;　後の余裕(0 - 120 sec)  
premium_post_margin = 80  
-------------------------------------------------  
radiko (392行付近)  
  
; 録音開始 : 番組開始時間 + deley - pre_mergin  
; 録音終了 : 番組終了時間 + deley + post_mergin  
;  
;　配信遅れ(0 - 120 sec)  
radiko_delay = 40  
;　前の余裕(0 - 120 sec)  
radiko_pre_margin = 25  
;　後の余裕(0 - 120 sec)  
radiko_post_margin = 80  
-------------------------------------------------  
らじる (467行付近)  
  
; 録音開始 : 番組開始時間 + deley - pre_mergin  
; 録音終了 : 番組終了時間 + deley + post_mergin  
;  
;　配信遅れ(0 - 120 sec)  
radiru_delay = 45  
;　前の余裕(0 - 120 sec)  
radiru_pre_margin = 25  
;　後の余裕(0 -120 sec)  
radiru_post_margin = 25  
-------------------------------------------------  

これで、ほぼ大丈夫だと思います。  
もっと余裕を持たせたい方は、post_marginを大きくしてください。    
ただし、放送後の無駄な部分が増えます。  
  
### 20.3 その他
  
局によって配信遅れにばらつきがあり、総務省の資料によると  
2024年4月現在、約32～約100秒の遅延があるようです。

全局に対応するためには、post_marginを多めにしたほうがいいかもしれません。  
私が経験したものとしては、ほかは1分くらいなのに、東京FMが2分近く遅れていました。
